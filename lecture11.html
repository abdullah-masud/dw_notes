01

What is Graph Data Science?

📖 Definition

Graph Data Science (GDS) is using graph algorithms to answer business-critical questions and make predictions from graph data. Neo4j has a built-in Graph Data Science library with hundreds of ready-to-use algorithms.

Real-world applications

  • Google Maps: Given your start (UWA) and destination (King's Park), GDS algorithms find all possible routes and suggest the shortest or fastest one based on traffic.
  • Robotics: A robot arm needs to move from point A to point B without hitting obstacles. GDS finds a safe path through the space.
  • Pac-Man AI: The game computer uses search algorithms to explore all possible moves and find a path that eats dots without being caught by ghosts.
  • Fraud detection: Tracing suspicious connected transaction patterns in financial networks.

The GDS Algorithm Categories in this Unit

Algorithm Type Key Use
BFS (Breadth First Search) Graph traversal Shortest path (unweighted), level-by-level exploration
DFS (Depth First Search) Graph traversal General graph exploration, visiting all nodes
A* (A-Star) Pathfinding Shortest path with weights + heuristic estimate
Minimum Spanning Tree Graph optimisation Find minimum-cost connection of all nodes
Random Walk Sampling Explore graphs randomly, graph embeddings, similarity
02

What is Search?

📖 Definition

Search in graph data science is how a computer tries to find different options to find the best answer to a problem. A solution is a path from the start node to the goal node.

Two types of search goals

  • General discovery: Explore what's inside the graph — visit all nodes, understand the structure. (Used by DFS)
  • Explicit search: Find a specific target — "find the shortest path from A to B." (Used by BFS, A*)
💡 Important Note

Graph search algorithms carve paths through the graph, but these paths may NOT necessarily be the computationally optimal path. BFS and DFS use equal weights (every edge = cost 1). Only A* uses actual edge weights, making it smarter for weighted graphs.

03

BFS — Breadth First Search

📖 Definition

BFS starts from a chosen node and explores ALL of its direct neighbours (1 hop away) before visiting neighbours 2 hops away, then 3 hops, and so on. It explores the graph level by level.

🎯 BFS Key Properties — Must Know

Unweighted: Every edge costs 1.
Queue = FIFO: First In, First Out. The node that was added first to the queue is processed first.
Finds shortest path: Because it explores level by level, the first time you reach a node IS always the shortest path.
Visits nearest neighbours first.

BFS Step-by-Step: How it works

Think of BFS like ripples in water — starting from your stone-drop point, it reaches all spots 1 step away, then 2 steps, then 3 steps, and so on.

EXAMPLE GRAPH — BFS from node A

A
B
C
D
E
F
G
BFS Visit Order starting from A:
Step Queue (before processing) Process Add to Queue
1 [A] A B, C
2 [B, C] B D, E
3 [C, D, E] C F, G
4 [D, E, F, G] D (none)
5 [E, F, G] E (none)
6 [F, G] F (none)
7 [G] G (none)

Final visit order: A → B → C → D → E → F → G

BFS visits level 1 (B,C) before level 2 (D,E,F,G). This is why it finds shortest paths.

BFS for Shortest Path — Worked Example

Starting from node 0 in the graph: 0 connects to 1 and 3. 1 connects to 2. 3 connects to 4 and 5. 4 connects to 6 and 7. 5 connects to 6.

Step Queue (FIFO) Process Neighbours (unvisited)
1 [0] 0 1, 3 → add to queue
2 [1, 3] 1 2 → add
3 [3, 2] 3 4, 5 → add
4 [2, 4, 5] 2 none
5 [4, 5] 4 6, 7 → add (3 already visited)
6 [5, 6, 7] 5 6 (already in queue)
7-8 [6, 7] 6, 7 none left

Visit order: 0 → 1 → 3 → 2 → 4 → 5 → 6 → 7

Shortest path from 0 to 7: 0 → 4 → 7 (cost = 3 hops). Why? BFS reaches node 7 via 0→3→4→7, which is the first (shortest) path found.

BFS uses cost = 1 per hop. The first time you reach any node = shortest path.

04

DFS — Depth First Search

📖 Definition

DFS starts from a chosen node and picks one neighbour, then goes as deep as possible along that branch before backtracking and trying the next option. It explores one branch all the way down before trying another.

🎯 DFS Key Properties — Must Know

Unweighted: Every edge costs 1.
Stack = LIFO: Last In, First Out. The most recently added node is processed next (like a stack of plates).
NOT for shortest path: DFS is for graph exploration/traversal, not shortest path finding.
When multiple neighbours: Visit in alphabetical/numerical order.

DFS Step-by-Step: How it works

Think of DFS like exploring a maze — you always keep going forward until you hit a dead end, then backtrack and try a different branch.

Simpler DFS Example

Graph with nodes 0,1,2,3,4. 0 connects to 1 and 2. 1 connects to 2. 2 connects to 3 and 4. Find visit order from 0.

Stack (LIFO) Process Add to Stack
[0] 0 1, 2 → stack = [2, 1] (1 on top — processed first numerically)
[2, 1] 1 2 already visited → nothing
[2] 2 3, 4 → stack = [4, 3]
[4, 3] 3 none
[4] 4 none

Visit order: 0 → 1 → 2 → 3 → 4

Each node visited at most once. LIFO stack means the most recently discovered node is explored next.

05

BFS vs. DFS — Side-by-Side Comparison

BFS — Breadth First Search
  • Data structure: Queue (FIFO — First In, First Out)
  • Explores: All nodes at current hop-distance before going deeper
  • Finds shortest path: ✅ Yes — because it reaches each node via minimum hops
  • Weighted: ❌ No — all edges cost 1
  • Best for: Finding shortest paths, level-by-level exploration, closeness centrality
  • Memory: Uses more memory (stores all nodes at current level)
  • Analogy: Ripples in water spreading outward
DFS — Depth First Search
  • Data structure: Stack (LIFO — Last In, First Out)
  • Explores: One branch as deeply as possible before backtracking
  • Finds shortest path: ❌ No — goes deep, not shortest
  • Weighted: ❌ No — all edges cost 1
  • Best for: General graph exploration, checking if a path exists, visiting all nodes
  • Memory: Uses less memory (only stores current path)
  • Analogy: Exploring a maze — go as far as possible, then backtrack
Feature BFS DFS A*
Data structure Queue (FIFO) Stack (LIFO) Priority queue
Weighted? No (cost=1 per hop) No (cost=1 per hop) Yes (uses edge weights)
Finds shortest path? Yes No Yes (optimal)
Uses heuristic? No No Yes — f(n)=g(n)+h(n)
Main use Shortest path in unweighted graphs Full graph exploration Shortest path in weighted graphs

06

A* Search — Weighted Pathfinding with a Heuristic

📖 Why A* Exists

BFS and DFS treat all edges as cost 1 and have no information about where the goal is. A* is smarter — it uses actual edge weights AND a heuristic estimate of remaining distance to the goal, so it focuses exploration toward the most promising paths.

The A* Formula — f(n) = g(n) + h(n)

f(n) = g(n) + h(n)
g(n)Actual cost from START to current node n along the path taken so far
h(n)Heuristic estimate of cost from node n to the GOAL. This is a "guess" — not the actual cost.
f(n)Total estimated cost of path through n. A* always expands the node with the LOWEST f(n).
💡 In Plain English

g(n) = "how much have I already spent to get here?" h(n) = "how much do I estimate it will cost to reach the goal from here?" f(n) = "what's my best guess at the total trip cost if I go through this node?" A* always picks the node with the smallest f(n) next — it prioritises paths that look cheapest overall.

What is a Heuristic?

  • A heuristic is a function that estimates how close a node is to the goal — it's a "smart guess"
  • Must be designed for the specific problem type
  • Common heuristics: Manhattan distance (grid-based movement, horizontal+vertical only), Euclidean distance (straight-line/crow-flies distance)
  • In the exam, heuristic values are given to you — you don't need to calculate them from scratch
  • The goal node always has h(n) = 0 (no estimated remaining distance when you're at the goal)

A* Worked Example — Full Step-by-Step

Find the shortest path from a to z. Numbers inside nodes = heuristic values h(n). Numbers on edges = actual costs.

Graph: a(h=11) --5-- b(h=8) --5-- d(h=4) --6-- z(h=0)
       a --4-- c(h=8) --1-- (c is node with low edge to b and e)
       c --10-- e(h=2) --5-- z
       b --8-- e
Step Current node Options f = g + h Choose
1 Start: a a→b: g=5, h=8 → f=13
a→c: g=4, h=8 → f=12		 13 vs 12 a→c (f=12 lowest)
2 At c a→c→d: g=4+1=5? (via c's edge to b, then d)? Check options
a→c→e: g=4+10=14, h=2 → f=16
Also explore a→b: f=13		 13 vs 16 a→b (f=13 lowest)
3 At b a→b→d: g=5+5=10, h=4 → f=14
a→b→c: but c visited with lower g		 14 a→b→d (f=14)
4 At d a→b→d→z: g=10+6=16, h=0 → f=16 16 REACHED GOAL

Shortest path: a → b → d → z

Total actual cost: 5 + 5 + 6 = 16 (heuristics are NOT added to the final cost — only edge costs)

Key rule: at each step, pick the option with the lowest f(n). The heuristic guides you toward the goal but is NOT included in the final cost calculation.

⚠️ Critical Exam Point — Heuristic is NOT part of final cost

The heuristic h(n) is used ONLY to decide which path to explore next. When you finally reach the goal and report the total cost, you only add up the actual edge costs g(n). Never include h(n) in the final answer.

Always expand the node with the lowest f(n) value across ALL current open options, regardless of how many steps you've already taken.

Minimum Spanning Tree (MST)

📖 Key Definitions

Tree: An undirected network where any two nodes are connected by exactly ONE path. No loops/cycles.
Spanning Tree: A tree that contains ALL nodes of the graph, connected with minimum edges (no cycles).
Minimum Spanning Tree (MST): The spanning tree with the minimum total edge weight — the cheapest way to connect all nodes.

Real-world use

The classic example: you need to lay cable/pipe/wire to connect N cities. You want every city reachable from every other, but you want to minimise the total cable used. The MST gives you the cheapest network.

Original network (has loops)

A ——4—— B ——6—— C
  \
   2
    \
     D ——3—— B
      \
       5
        \
         C

Edges: A-B(4), B-C(6), A-D(2), D-B(3), D-C(5), B-D(10)

MST (no loops, minimum cost)

Keep edges: A-D (2), D-B (3), D-C (5), A-B (4)
Drop: B-C (6), B-D (10) ← too expensive

Total cost of MST = 2 + 3 + 4 + 5 = 14
Dropped the 10 and 6 weight edges

MST includes all nodes, uses minimum edge cost, and has no cycles. Left = original graph with loops. Right = MST (tree, no loops).

Prim's Algorithm — How to Find the MST

📖 Prim's Algorithm Steps

1. Start with any chosen node.
2. Find the cheapest edge connecting ANY node already in your tree to ANY node NOT yet in the tree.
3. Add that edge and the new node to the tree.
4. Repeat steps 2–3 until ALL nodes are in the tree.
5. The result is the MST.

Prim's — Worked Example (from slides)

Graph with nodes a,b,c,d,e,f,g. Find MST starting from a.

Tree so far Cheapest edge to new node Add to MST Cost so far
{a} a-b (4) vs a-c (8) vs a-d (8) a-b (weight 4) 4
{a,b} b-c (8), b-d (8) vs a-c (8), etc. — check min from any node in tree b-d (weight 2) if exists, or next cheapest 4+2=6
{a,b,d} Find cheapest to f or e or c Pick lowest cost +next min
Continue until all nodes included. Total cost = 22 for this example.
Key property: The MST total weight is always unique (always the same minimum), but the MST structure itself may not be unique — when edges have equal weights, there can be multiple valid MSTs (but same total cost).

Prim's algorithm from the slides: 2 MSTs exist for that example graph (because some edges have equal weight) but both have total cost = 22.

🎯 Exam Trick — Multiple MSTs Are OK

If two edges have the same weight and you could pick either, the question might have multiple valid MSTs. This is fine — the MST structure may differ but the total weight is always the same. If asked to find the MST, show your work step by step and justify each edge choice.

08

Random Walk

📖 Definition

A random walk starts from a node and moves to a connected node chosen at random, repeatedly, for a specified number of steps or until a specified end node is reached. There is no "smart" decision — movement is random.

The UWA Analogy (from the lecturer)

"Imagine you are a new student at UWA and someone tells you to go to the CSSE building, but you have no map. You don't know the layout. So you just start walking randomly — maybe you wander to the Business School, then Engineering, then accidentally arrive at CSSE. That's a random walk."

Why would we use something so seemingly useless?

  • Graph embeddings: Translate the graph structure into numerical coordinates (vectors) that computers can process with machine learning algorithms
  • Similarity measurement: Two words like "happy" and "joyful" — random walks can measure how similarly they are connected in a language graph
  • Network analysis: Understand the structure of complex networks without prior knowledge of them
  • Training data: Generate training data for graph machine learning models (e.g. Node2Vec algorithm uses random walks)

Random Walk Parameters

  • walkLength: How many steps to take (e.g. 5 means visit 5 nodes from the start)
  • walksPerNode: How many separate random walks to perform from the same starting node
  • No calculation needed in exam — just understand the concept

Projected Graphs — Required for Neo4j GDS

📖 What is a Projected Graph?

Before running ANY GDS algorithm in Neo4j, you must first project your graph. A projected graph is an in-memory copy of (part of) your Neo4j database, loaded into the GDS library for analysis. It is NOT stored permanently — it's like a temporary view.

Why do we need to project?

Your full graph database might have many node types (Person, Movie, Company, etc.) and relationship types. GDS algorithms usually only need a subset. Projection lets you:

  • Choose which node labels to include (e.g. only Place nodes)
  • Choose which relationship types to include (e.g. only ROAD relationships)
  • Control graph orientation (directed or undirected)
  • Keep the analysis in memory (faster, doesn't affect the real database)

Creating a projected graph

gds.graph.project — project the Transport graph
// Project: include all Place nodes and EROAD relationships CALL gds.graph.project('Transport', 'Place', 'EROAD') // 'Transport' = project name (used later when calling algorithms) // 'Place' = node label to include // 'EROAD' = relationship type to include // Default orientation = NATURAL (same as the stored graph)
⚠️ Important — Different Algorithms Need Different Projects

Some algorithms (BFS, DFS) work with the same projected graph. A* requires different node properties (latitude, longitude) so you may need to delete the old project and create a new one. Always check that your project includes the properties your algorithm needs.

gds.dfs.stream — DFS traversal
MATCH (source:Place{name:"Ipswich"}), (target:Place{name:'Den Haag'}) CALL gds.dfs.stream('Transport', {sourceNode:source, targetNodes:target}) YIELD path RETURN path

A* Shortest Path in Neo4j GDS

gds.shortestPath.astar.stream — weighted shortest path
MATCH (source:Place {name: 'Ipswich'}), (target:Place {name: 'Utrecht'}) CALL gds.shortestPath.astar.stream('Transport', { sourceNode: source, targetNode: target, latitudeProperty: 'latitude', // used as heuristic longitudeProperty: 'longitude', // used as heuristic relationshipWeightProperty: 'distance' }) YIELD index, sourceNode, targetNode, totalCost, nodeIds, costs, path RETURN gds.util.asNode(sourceNode).name AS sourceNodeName, gds.util.asNode(targetNode).name AS targetNodeName, totalCost, [nodeId IN nodeIds | gds.util.asNode(nodeId).name] AS nodeNames, costs ORDER BY index
⚠️ A* GDS Returns Only Nodes — No Relationships

The path returned by GDS algorithms is a virtual path with nodes only — no relationship arrows. This is because GDS operates on the in-memory projected graph, which doesn't have direct references to the real Neo4j relationship objects.

Minimum Spanning Tree in Neo4j GDS

gds.beta.spanningTree.write — compute and write MST
MATCH (n:Place{name: 'Ipswich'}) CALL gds.beta.spanningTree.write( 'Transport', {sourceNode: id(n), relationshipWeightProperty: 'distance', writeProperty: 'writeCost', writeRelationshipType: 'MINST'} // creates MINST relationships ) YIELD preProcessingMillis, computeMillis, writeMillis, effectiveNodeCount RETURN preProcessingMillis, computeMillis, writeMillis, effectiveNodeCount // Then display the MST: MATCH (p1:Place)-[m:MINST]->(p2:Place) RETURN p1, m, p2

Random Walk in Neo4j GDS

gds.randomWalk.stream — random walk from London
MATCH (source:Place {name: "London"}) CALL gds.randomWalk.stream('Transport', { sourceNodes: [id(source)], walkLength: 5, // take 5 steps walksPerNode: 1 // do 1 walk from London }) YIELD nodeIds, path RETURN nodeIds, [nodeId IN nodeIds | gds.util.asNode(nodeId).name] AS nodeNames, path

Summary — The Exam Cheat Sheet

Lecture 11 — All Algorithm Properties (from slides summary slide)

  • BFS: Unweighted (cost=1 per edge) · Queue = FIFO · Finds shortest path · Explores level by level
  • DFS: Unweighted (cost=1 per edge) · Stack = LIFO · Does NOT find shortest path · For graph exploration/traversal

Key Definitions

  • g(n) = actual cost from start to node n along path taken
  • h(n) = heuristic estimate from node n to goal (always given in exam, never computed)
  • f(n) = g(n) + h(n) = total estimated cost through n (the lower, the better)
  • Heuristic for A*: goal node always h=0. Common: Manhattan distance, Euclidean distance.
  • Projected graph = in-memory copy of your graph for GDS algorithms. Created with gds.graph.project(). Temporary only.
  • GDS path result = virtual path (nodes only, no relationship objects) because GDS runs on in-memory projected graph.

Exam-Style Practice Questions

Use BFS starting from node 0 to find: (a) the visit order, (b) the shortest path from 0 to 6. Graph: 0→1, 0→2, 1→3, 1→4, 2→5, 2→6.

Hint: (a) BFS queue FIFO: Start [0] → process 0 → add 1,2 → process 1 → add 3,4 → process 2 → add 5,6 → process 3,4,5,6. Visit order: 0,1,2,3,4,5,6. (b) Shortest path 0→2→6, cost = 2 hops.

Use DFS starting from node A to find the visit order. Graph: A connects to B and C. B connects to D and E. C connects to F. D connects to G. Alphabetical order for tie-breaking.

Hint: Stack LIFO, alphabetical order. Start [A] → process A, add B,C (stack [C,B]) → process B, add D,E (stack [C,E,D]) → process D, add G (stack [C,E,G]) → process G (stack [C,E]) → process E (stack [C]) → process C, add F (stack [F]) → process F. Visit order: A,B,D,G,E,C,F.

Explain the difference between BFS and DFS. For which problems would you use each? What data structure does each use?

Hint: BFS = queue (FIFO), level-by-level, finds shortest path in unweighted graphs. DFS = stack (LIFO), deep first, used for exploring all nodes (not shortest path). Use BFS when you need shortest path. Use DFS when you need to explore the full graph structure.

Use A* to find the shortest path from S to G. Graph: S→A (cost 2, h=4), S→B (cost 5, h=2), A→G (cost 6, h=0), B→G (cost 3, h=0). Goal node G has h=0.

Hint: Step 1: S→A: f=2+4=6, S→B: f=5+2=7. Choose S→A (f=6 lower). Step 2: From A: S→A→G: g=2+6=8, h=0, f=8. From B: f=7. Choose B (f=7 lower). Step 3: S→B→G: g=5+3=8, h=0, f=8. Tie between S→A→G and S→B→G both cost 8. Both are valid shortest paths with total cost 8.

What is a Minimum Spanning Tree? Describe Prim's algorithm in plain steps and explain why the total weight is always unique even if the MST structure is not.

Hint: MST = spanning tree (connects all nodes, no loops) with minimum total edge weight. Prim's: start from any node, repeatedly add the cheapest edge connecting a new node to any already-in-tree node, until all nodes included. Weight unique because MST is optimal — but when edges have equal weights, there may be multiple ways to achieve that same minimum.

In A*, a student adds the heuristic value h(n) to the final path cost to get the total. Are they correct? Explain.

Hint: NO — incorrect. Heuristics h(n) are used ONLY for deciding which node to explore next. They are never added to the final cost. The total path cost = sum of actual edge weights g(n) along the chosen path only. The heuristic is an estimation tool, not a real distance.

What is a projected graph in Neo4j GDS? Why is it needed before running BFS, DFS, or A*?

Hint: A projected graph is a temporary in-memory copy of a subset of the Neo4j database (selected node labels + relationship types). It's needed because GDS algorithms operate on this in-memory structure, not the actual stored database.
